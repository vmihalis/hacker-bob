# Hacker Bob v1.1.0 Release Notes

## Highlights

`v1.1.0` is a community-quality polish release focused on install confidence, clean removal, and package-name migration. The canonical npm package is now `hacker-bob`, and `hacker-bob-cc` remains as a small compatibility wrapper that delegates to the canonical CLI.

## Install

Use the canonical package for one-off installs:

```bash
npx -y hacker-bob@1.1.0 install /path/to/your/project
```

Or install the CLI globally:

```bash
npm install -g hacker-bob@1.1.0
hacker-bob install /path/to/your/project
```

The compatibility package is also published:

```bash
npm install -g hacker-bob-cc@1.1.0
```

Packages:

- `hacker-bob`: https://www.npmjs.com/package/hacker-bob/v/1.1.0
- `hacker-bob-cc`: https://www.npmjs.com/package/hacker-bob-cc/v/1.1.0

## CLI

- Added `--adapter claude|codex|generic-mcp|all` for install, update, doctor, and uninstall.
- Added `hacker-bob doctor <project-dir> [--json]` for read-only install diagnostics.
- Added `hacker-bob uninstall <project-dir> [--dry-run] [--yes] [--json]` for conservative removal of Bob-managed files and config entries.
- Updated help text to show install, update, check-update, doctor, uninstall, and one-project-per-install semantics.

## Packaging

- Bumped the canonical `hacker-bob` package to `1.1.0`.
- Added the `packages/hacker-bob-cc` compatibility package at `1.1.0`.
- Included the neutral resources, Claude adapter, Codex plugin, generic MCP prompt docs, MCP runtime, CLI, and migration helpers in the canonical package.
- Updated the GitHub tag release workflow to publish the canonical package first and the compatibility package second, both with npm provenance.

## Documentation

- Added a Quickstart to the README.
- Added troubleshooting docs with doctor usage and common install/update fixes.
- Updated the bug report template to ask for sanitized doctor JSON output.
