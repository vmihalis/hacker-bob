# Hacker Bob v1.1.1 Release Notes

## Highlights

`v1.1.1` is a slash-menu hygiene patch. The Claude Code slash menu used to surface two parallel sets of Bob entry points — `/bob-hunt` and `/bob:hunt` (and the same for status/debug) — because the skills and the command shims advertised the same workflow under different names. The skills now declare colon-form `name:` frontmatter and back the `/bob:` slash commands directly, the redundant shims are gone, and the installer proactively cleans them up on upgrade so users coming from `<=1.1.0` end up with a single, deduplicated menu.

## Install

```bash
npx -y hacker-bob-cc@1.1.1 install /path/to/your/project
```

Or install the CLI globally:

```bash
npm install -g hacker-bob-cc@1.1.1
hacker-bob install /path/to/your/project
```

The convenience alias is also published:

```bash
npm install -g hacker-bob@1.1.1
```

Packages:

- `hacker-bob-cc`: https://www.npmjs.com/package/hacker-bob-cc/v/1.1.1
- `hacker-bob`: https://www.npmjs.com/package/hacker-bob/v/1.1.1

## Slash command de-duplication

- The three orchestrator skills now declare colon-form names: `bob:hunt`, `bob:status`, `bob:debug`. Claude Code resolves skills by the frontmatter `name:` rather than the directory, so the skill itself IS the slash command — no shim is required.
- `commands/bob/{hunt,status,debug}.md` were deleted from the source. The only remaining shim is `commands/bob/update.md`, because `/bob:update` is backed by an installer hook rather than a skill.
- The skill directory names (`bountyagent`, `bountyagentstatus`, `bountyagentdebug`) are unchanged for Windows path safety.

## Upgrade safety

- `scripts/install.js` and `dev-sync.sh` proactively delete the legacy `commands/bob/{hunt,status,debug}.md` shims on every install and upgrade. Without this step, a user upgrading from `1.1.0` would keep both the new colon-named skills and the orphan hyphen-named shims, re-introducing the duplicate menu entries.
- `scripts/lifecycle.js` keeps a `LEGACY_BOB_COMMAND_FILES` manifest so that `hacker-bob uninstall` still removes the deprecated shims from older installs while `doctor` only expects the current set.

## Behavior unchanged

- No changes to MCP server, hunt/verify/grade/report pipeline, hooks, agents, or recon flow.
- No changes to user-facing slash command syntax — `/bob:hunt`, `/bob:status`, `/bob:debug`, `/bob:update` continue to work exactly as documented.

## After upgrading

Restart Claude Code in the target project once the new version is installed. The legacy `/bob-hunt`, `/bob-status`, `/bob-debug` entries will disappear from the slash menu and only the colon-form entries will remain.
