# Hacker Bob v1.1.4 Release Notes

## Highlights

`v1.1.4` fixes the install path for the policy replay harness introduced in `v1.1.3`.

The `v1.1.3` npm package included `testing/policy-replay/`, but the installer only copied `.claude/` and `mcp/` runtime files into the target project. That meant the updated `/bob-debug` skill could reference `testing/policy-replay/replay.mjs` and `tune.mjs` in an installed workspace where those files were missing.

## Fix

The installer now copies `testing/policy-replay/` into the target project and skips any nested `node_modules` directory. `hacker-bob doctor` also verifies that the replay harness is present.

## Install

```bash
npx -y hacker-bob-cc@1.1.4 install /path/to/your/project
```

After installing, fully restart Claude Code in that project so `/bob-debug` reloads.

## Verification

- `npm test` passes.
- `npm run release:check` passes.
- Install smoke and CLI doctor tests cover the policy replay harness files.
