# Hacker Bob v1.1.2 Release Notes

## Highlights

`v1.1.2` finishes what `v1.1.1` set out to do: one slash command per Bob skill, no duplicates. The `v1.1.1` release tried to dedupe the menu by writing colon-form `name:` frontmatter (`bob:hunt`, `bob:status`, `bob:debug`) into each skill, but Claude Code v2.1.119 rejects colons in skill `name:` (the field accepts lowercase letters, numbers, and hyphens only) and silently fell back to the directory name. Result: typing `/bob:hunt` got rewritten to `/bountyagent` on enter, and the slash menu still showed the same legacy entries.

`v1.1.2` solves the underlying problem by renaming the skill directories and frontmatter to hyphen form so directory, frontmatter, and slash all agree:

- `.claude/skills/bountyagent/` → `.claude/skills/bob-hunt/` with `name: bob-hunt`
- `.claude/skills/bountyagentstatus/` → `.claude/skills/bob-status/` with `name: bob-status`
- `.claude/skills/bountyagentdebug/` → `.claude/skills/bob-debug/` with `name: bob-debug`

The `/bob:update` command is renamed to `/bob-update` for consistency, and its file moves from `.claude/commands/bob/update.md` to `.claude/commands/bob-update.md` so all four Bob slashes share the same hyphen scheme.

## User-facing changes

- `/bob-hunt target.com` instead of `/bob:hunt target.com`
- `/bob-status` instead of `/bob:status`
- `/bob-debug` instead of `/bob:debug`
- `/bob-update` instead of `/bob:update`

The MCP server name (`bountyagent`) and every `mcp__bountyagent__*` tool name is unchanged — only the user-facing slash menu surface moves.

## Install

```bash
npx -y hacker-bob-cc@1.1.2 install /path/to/your/project
```

After the install, fully restart Claude Code in that project. The slash menu should show `/bob-hunt`, `/bob-status`, `/bob-debug`, `/bob-update` — one entry per command, no `bountyagent*` or `bob:*` ghosts.

## Upgrade safety

The installer proactively deletes the legacy paths so upgrades from `<=1.1.1` self-clean:

- `.claude/skills/bountyagent`, `.claude/skills/bountyagentstatus`, `.claude/skills/bountyagentdebug` — wiped
- `.claude/commands/bob/hunt.md`, `status.md`, `debug.md`, `update.md` — wiped, plus the now-empty `commands/bob/` directory

The uninstall manifest sweeps the new layout, the `v1.1.1` layout, and the `v1.1.0` layout, so existing installs at any point in that range still clean up entirely.
