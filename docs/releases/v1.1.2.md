# Hacker Bob v1.1.2 Release Notes

## Highlights

`v1.1.2` finishes the Claude slash-menu cleanup with one visible entry per Bob workflow. Claude skills now use hyphen names because Claude Code accepts lowercase letters, numbers, and hyphens in skill `name:` frontmatter.

## Claude User-Facing Changes

- `/bob-hunt target.com` instead of `/bob:hunt target.com`
- `/bob-status` instead of `/bob:status`
- `/bob-debug` instead of `/bob:debug`
- `/bob-update` instead of `/bob:update`

The MCP server name (`bountyagent`) and every `mcp__bountyagent__*` tool name is unchanged.

Codex syntax is unchanged and remains `$hacker-bob:hunt`, `$hacker-bob:status`, `$hacker-bob:debug`, and `$hacker-bob:update`.

## File Layout

- `.claude/skills/bountyagent/` -> `.claude/skills/bob-hunt/`
- `.claude/skills/bountyagentstatus/` -> `.claude/skills/bob-status/`
- `.claude/skills/bountyagentdebug/` -> `.claude/skills/bob-debug/`
- `.claude/commands/bob/update.md` -> `.claude/commands/bob-update.md`

## Upgrade Safety

The installer removes legacy Claude paths on upgrade:

- `.claude/commands/bob/hunt.md`
- `.claude/commands/bob/status.md`
- `.claude/commands/bob/debug.md`
- `.claude/commands/bob/update.md`
- `.claude/skills/bountyagent`
- `.claude/skills/bountyagentstatus`
- `.claude/skills/bountyagentdebug`

After installing or updating, fully restart Claude Code in the target project so it reloads skills, commands, MCP config, hooks, and statusline setup.
