# Code of Conduct

## Our standards

This project welcomes practical, good-faith contributions from people with
different backgrounds and experience levels. Keep discussion focused on the
work, explain technical disagreement clearly, and assume public threads may be
read by new users trying to learn.

Examples of welcome behavior:

- Giving specific, actionable feedback.
- Asking clarifying questions before making assumptions.
- Respecting authorization boundaries for security testing.
- Avoiding disclosure of private target data, credentials, or unpublished
  third-party vulnerabilities.

Examples of unacceptable behavior:

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing another person's private information.
- Pressuring contributors to run unauthorized scans or disclose private reports.
- Repeated low-effort disruption after maintainers ask for a change in behavior.

## Enforcement

Maintainers may edit or delete comments, close issues or pull requests, block
users, or report abuse to GitHub when needed to keep the project usable and safe.

If you see harmful behavior, notify the maintainers in the relevant thread when
that is safe. For serious abuse or threats, use GitHub's abuse reporting tools.
Security vulnerabilities in Hacker Bob itself should be reported through
`SECURITY.md`, not through conduct threads.
